## 四、如何修改测试数据？

### 修改边界值数据

```yaml
# data/boundary_test_data.yaml

username:
  boundaries:
    - value: ""           # 修改这里
      expected_valid: false
      description: "空字符串"
    
    - value: "a" * 50     # 添加新的边界值
      expected_valid: false
      description: "50字符"
```

### 修改异常测试数据

```yaml
# data/invalid_test_data.yaml

sql_injection:
  payloads:
    - value: "'; DROP TABLE users; --"
      description: "DROP TABLE 注入"
    
    - value: "1; DELETE FROM users WHERE 1=1"  # 添加新的注入
      description: "DELETE 注入"
```

### 修改 API 测试数据

```yaml
# data/api_test_data.yaml

users:
  valid:
    - username: "octocat"
      expected_id: 583231
    
    - username: "newuser"     # 添加新用户
      expected_id: 123456
```

### 修改 Web UI 测试数据

```yaml
# data/web_test_data.yaml

login:
  valid:
    - username: "testuser"
      password: "testpassword"
    
    - username: "admin"       # 添加新的有效登录
      password: "admin123"
```

### 修改压力测试配置

```yaml
# data/framework_test_data.yaml

stress_test:
  high_concurrency:
    concurrent_requests: 200    # 增加并发数
    expected_success_rate: 100
```

## 七、总结

| 想做什么 | 去哪里 | 用什么 |
|----------|--------|--------|
| 测试正常流程 | 测试文件 | `DataGenerator.generate_xxx()` |
| 测试边界值 | `data/boundary_test_data.yaml` | `DataHelper.load_boundary_data()` |
| 测试异常场景 | `data/invalid_test_data.yaml` | `DataHelper.load_invalid_data()` |
| 测试特定 API | `data/api_test_data.yaml` | `DataHelper.load_api_data()` |
| 测试 Web UI | `data/web_test_data.yaml` | `DataHelper.load_web_data()` |
| 测试框架组件 | `data/framework_test_data.yaml` | `DataHelper.load_framework_data()` |
| 压力/负载测试 | `data/framework_test_data.yaml` | `DataHelper.load_framework_data()['stress_test']` |
| 需要可重复数据 | 测试文件 | `DataGenerator.set_seed(12345)` |
