# 插件系统指南

## 一、简介

本项目采用 **"硬逻辑 + 软钩子"** 组合模式的插件系统，参考了 Pytest、Scrapy 等顶级框架的设计理念。

### 核心设计理念

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                     "硬逻辑 + 软钩子" 组合模式                               │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   核心插件（Core Plugin）                                                    │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  - 代码逻辑在 common/ 核心库中                                       │  │
│   │  - 默认强制加载，不可禁用                                            │  │
│   │  - 实现与普通插件相同的接口                                          │  │
│   │  - 示例：熔断器、限流器                                              │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   普通插件（Normal Plugin）                                                  │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  - 可选功能，可自由启用/禁用                                         │  │
│   │  - 用户可自定义扩展                                                  │  │
│   │  - 示例：日志、指标、缓存                                            │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 为什么这样设计？

| 特性 | 核心插件 | 普通插件 |
|------|----------|----------|
| **必要性** | 系统必需 | 可选增强 |
| **禁用后果** | 系统可能崩溃/被封禁 | 只是缺少某些功能 |
| **调用位置** | 请求最底层 | 请求前后钩子 |
| **性能要求** | 极高（每次请求） | 较低 |
| **设计目标** | 稳定可靠 | 灵活扩展 |

## 三、核心接口

### 3.1 插件基类

```python
from common.plugin_system import Plugin, PluginType

class MyPlugin(Plugin):
    """自定义插件"""
    
    @property
    def name(self) -> str:
        """插件名称（必须实现）"""
        return "my_plugin"
    
    @property
    def version(self) -> str:
        """插件版本（必须实现）"""
        return "1.0.0"
    
    @property
    def description(self) -> str:
        """插件描述"""
        return "这是一个自定义插件"
    
    @property
    def plugin_type(self) -> PluginType:
        """插件类型，默认为 NORMAL"""
        return PluginType.NORMAL
    
    # 生命周期钩子
    def on_load(self):
        """插件加载时调用"""
        pass
    
    def on_enable(self):
        """插件启用时调用"""
        pass
    
    def on_disable(self):
        """插件禁用时调用"""
        pass
    
    def on_unload(self):
        """插件卸载时调用"""
        pass
    
    # 请求钩子
    def before_request(self, method: str, url: str, **kwargs):
        """请求前钩子，返回非 None 则跳过实际请求"""
        pass
    
    def after_request(self, response, method: str, url: str, **kwargs):
        """请求后钩子"""
        pass
    
    def on_request_error(self, error: Exception, method: str, url: str, **kwargs):
        """请求错误钩子"""
        pass
    
    # 测试钩子
    def before_test(self, test_name: str):
        """测试前钩子"""
        pass
    
    def after_test(self, test_name: str, result):
        """测试后钩子"""
        pass
    
    def on_test_failure(self, test_name: str, error):
        """测试失败钩子"""
        pass
    
    def on_test_success(self, test_name: str):
        """测试成功钩子"""
        pass
```

### 3.3 插件管理器

```python
from common.plugin_system import get_plugin_manager

# 获取全局插件管理器
manager = get_plugin_manager()

# 注册插件
manager.register(MyPlugin())

# 获取插件
plugin = manager.get_plugin('my_plugin')

# 启用/禁用插件（核心插件不可禁用）
manager.enable('my_plugin')
manager.disable('my_plugin')

# 获取插件列表
all_plugins = manager.get_all_plugins()
core_plugins = manager.get_core_plugins()
normal_plugins = manager.get_normal_plugins()

# 执行钩子
manager.execute_hook('before_test', 'test_name')
```

## 四、如何使用

### 4.1 获取插件管理器

```python
from common.plugin_system import get_plugin_manager

# 获取全局实例（自动加载所有内置插件）
manager = get_plugin_manager()
```

### 4.2 注册自定义插件

```python
from common.plugin_system import Plugin, register_plugin

class MyCustomPlugin(Plugin):
    @property
    def name(self) -> str:
        return "my_custom"
    
    @property
    def version(self) -> str:
        return "1.0.0"
    
    def before_request(self, method: str, url: str, **kwargs):
        print(f"即将发送请求: {method} {url}")

# 方式1：使用便捷函数
register_plugin(MyCustomPlugin())

# 方式2：使用管理器
manager = get_plugin_manager()
manager.register(MyCustomPlugin())
```

### 4.3 在请求中使用

```python
from common.base_requests import BaseRequests

# 插件系统自动集成
client = BaseRequests(enable_plugins=True)

# 请求会自动触发插件钩子
response = client.get("/api/users")
```

### 4.4 在测试中使用

```python
# conftest.py 中已自动集成
# 测试前后会自动触发 before_test/after_test 钩子

def test_example():
    # 测试代码...
    pass
```

## 七、自定义插件示例

### 7.1 通知插件

```python
from common.plugin_system import Plugin

class NotifyPlugin(Plugin):
    """测试完成后发送通知"""
    
    @property
    def name(self) -> str:
        return "notify"
    
    @property
    def version(self) -> str:
        return "1.0.0"
    
    def on_test_failure(self, test_name: str, error):
        """测试失败时发送通知"""
        self._send_notification(f"测试失败: {test_name}", error)
    
    def _send_notification(self, title: str, message: str):
        """发送通知（可对接企业微信、钉钉等）"""
        print(f"[通知] {title}: {message}")

# 注册插件
from common.plugin_system import register_plugin
register_plugin(NotifyPlugin())
```

## 九、最佳实践

1. **核心功能使用核心插件**：熔断、限流等关键功能应作为核心插件
2. **可选功能使用普通插件**：日志、指标、缓存等可选功能作为普通插件
3. **钩子要轻量**：钩子函数应尽快返回，避免影响性能
4. **异常处理**：钩子中的异常会被捕获并记录，不会影响主流程
5. **命名规范**：插件名称使用小写下划线格式，如 `my_plugin`
